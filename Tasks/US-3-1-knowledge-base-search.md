# Task: US-3.1 คลังความรู้สำหรับผู้ใช้

## เป้าหมาย
ช่วยให้ผู้ใช้ค้นหาคำตอบจากบทความก่อนสร้าง Ticket ลดภาระทีม IT

## รายละเอียดงาน
- สร้างหน้า Knowledge Base พร้อมช่องค้นหาและหมวดหมู่
- รองรับการจัดอันดับผลลัพธ์ (เช่น ตามความนิยม หรือตามคำค้น)
- แสดงบทความพร้อมเนื้อหา รูปภาพ หรือลิงก์ที่เกี่ยวข้อง
- วัด metrics เช่น จำนวนการเปิดบทความเพื่อใช้ในรายงาน

## เงื่อนไขสำเร็จ
- ผู้ใช้ค้นหาและเปิดบทความได้ภายใน 3 คลิกจากหน้าแรก
- ระบบต้องตอบสนองการค้นหาไม่เกิน 3 วินาที
- บทความที่ไม่มีผลลัพธ์ต้องแสดงข้อความแนะนำการแจ้ง Ticket
