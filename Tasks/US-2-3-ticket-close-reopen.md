# Task: US-2.3 ปิดงานหรือเปิดงานใหม่

## เป้าหมาย
ให้ User ยืนยันการแก้ไขหรือรีโอเพน Ticket ที่ยังไม่จบจากหน้ารายละเอียด

## รายละเอียดงาน
- เพิ่มปุ่ม `Close` และ `Re-open` พร้อมกลไกยืนยันก่อนเปลี่ยนสถานะ
- บันทึกบันทึกเหตุผลเมื่อ User เลือก Re-open
- อัปเดตสถานะใน workflow ตาม FR-4 และแจ้งเตือน Admin
- ปิด Ticket อัตโนมัติหลัง Resolved 3 วัน (batch job หรือ scheduled task)

## เงื่อนไขสำเร็จ
- ปุ่มทำงานเฉพาะ User เจ้าของ Ticket
- การปิดหรือเปิดใหม่สะท้อนในรายการและประวัติสถานะทันที
- มีหลักฐานว่าระบบปิดอัตโนมัติหลังครบเวลาตามกำหนด
