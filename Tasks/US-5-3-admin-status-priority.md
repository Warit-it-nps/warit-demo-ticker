# Task: US-5.3 ปรับสถานะและความสำคัญของ Ticket

## เป้าหมาย
ให้ Admin อัปเดตสถานะและ Priority ของ Ticket ได้ตาม workflow ที่กำหนด

## รายละเอียดงาน
- สร้าง UI ปรับสถานะ (Open, In Progress, Pending, Resolved, Closed, Canceled)
- ออกแบบการควบคุมเพื่อป้องกันการข้ามสถานะผิดลำดับ (เช่น ปิดงานได้เฉพาะ Resolved)
- อนุญาตให้ปรับ Priority พร้อมบันทึกเหตุผล
- บันทึกประวัติการเปลี่ยนแปลงพร้อมผู้แก้ไข

## เงื่อนไขสำเร็จ
- สถานะที่เลือกได้สอดคล้องกับ FR-4
- มีการตรวจสอบสิทธิ์ให้เฉพาะ Admin ที่รับผิดชอบหรือ Manager ปรับสถานะได้
- การเปลี่ยนสถานะแจ้งเตือนผู้ใช้งาน/ผู้เกี่ยวข้องตามที่กำหนด
