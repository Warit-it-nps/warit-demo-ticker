# Task: FR-4 Workflow สถานะ Ticket

## เป้าหมาย
กำหนดและบังคับใช้สถานะของ Ticket ให้เป็นไปตามมาตรฐานที่ระบุใน PRD

## รายละเอียดงาน
- ออกแบบ state machine สำหรับสถานะ Open, In Progress, Pending, Resolved, Closed, Canceled
- กำหนดเงื่อนไขในการเปลี่ยนสถานะแต่ละคู่ พร้อม validation และข้อความเตือน
- พัฒนางานเบื้องหลังสำหรับ auto-close หลัง Resolved 3 วัน
- บันทึกประวัติการเปลี่ยนสถานะพร้อมข้อมูลผู้ดำเนินการ

## เงื่อนไขสำเร็จ
- ระบบป้องกันการเปลี่ยนสถานะผิดลำดับ เช่น ปิดงานโดยยังไม่ Resolved
- Auto-close ทำงานและมี log ยืนยันทุกครั้ง
- ส่วนอื่นๆ (UI, API) สามารถ reuse workflow เดียวกันได้
